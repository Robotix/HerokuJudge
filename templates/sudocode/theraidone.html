Write a code to optimise your strategy to find and destroy enemy bunkers in a jungle.

Description

You work with the CIA and are running a military campaign against the terrorists. Your most recent undertaking is a mission to find and destroy enemy bunkers in the African jungles. Each bunker you have to attack has a number of cells, every one of which has to be attacked to destroy that bunker. Your intelligence has a list of the number of cells per bunker. You fly out with a helicopter to find and destroy these bunkers in the dense jungle. Your only option however to locate these bunkers is to bomb different spots randomly in the jungle until you hit one of the cells of a bunker. Write a strategic code that can help you minimise the number of bombs you use to destroy these bunkers.

Assumptions

The Jungle can be assumed to be spread out over a 2-d array. Each cell of a bunker occupies one pixel.
Cells can be assumed to be located adjacent to each other in a 4-connected connectivity fashion in the 2-d array.

Input

The first line would contain an integer n (10 <= n <= 500) denoting dimension of the square grid.
The second line would contain an integer m (m < n*n) denoting the total number of bunker cells in the square grid.

Event Rules

The jungle is assumed to be a grid of any size between 10*10 to 500*500.
The cells of one bunker can be found only in a 4-connected connectivity in the grid. So once you have found one cell in a bunker, you can rest assured that the next one will be on any of four the adjacent pixels.
The position of a cell in the grid is defined by a pair of space separated integers “r c” denoting the cell position with row number r and column number c.
Bombs are air-dropped on to a particular cell in the jungle.
Each query is a string consisting of two space-separated integers printed to the STDOUT. The first integer denotes the row number of the cell position and the second integer the column number respectively.
Both row number 'r' and column number 'c' are zero-indexed.
The participants code must send queries to determine if the bomb drop was successful.
The simulator counts the number of queries sent.
For each query, the simulator would either return the string “YES” or “NO” answering whether the cell position mentioned in the query contained a weak point to the participant’s STDIN. The participant’s code must read the result of the query as a string from the STDIN.

Scoring

The participant's code with the minimum number of queries wins.
Tie breakers will be resolved on the basis of minimum running time..

How are submissions compiled?

Your code is tested by a computer automatically, not by a human being, and you have to write your code accordingly.
The judge compiles the participant's code with the standard compilers for each language.
The following libraries are allowed to be imported in the participant's code:
Standard Math Library
OpenCV
Importing any other library would cause your submission to fail.

How are submissions executed?

The judge uses a sandboxed simulation environment to run the participant's code.
Deviation from expected problem solving procedures would result in a wrong submission. Repetition of such behaviour would result in the participant being banned.

What can my program output to the standard output?

The participant's program can only output queries to the standard output. The format of a query is:
<row-number>*space*<column-number>*newline-character*
For example: If the query is to the cell with row number 0 and column number 15, the corresponding query string would be:
"0 15\n"
NOTE:Failing to write a newline-character would not flush the output buffer. The program must explicitly flush the output buffer in such cases. Failure to do so may result in a wrong submission.
If the program outputs any random string other than a query, the submission would fail.

How does my input vary?

The first line of input would contain an integer n denoting the dimension of the square grid. The second line of input would contain an integer m denoting the total number of bunkers in the square grid. Every subsequent input would either be a string "YES\n" or "NO\n", the result to a query. Every query would have only one result.
