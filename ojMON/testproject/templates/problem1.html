Write a code to optimise your strategy to find and destroy enemy bunkers in a jungle.

Description

You work with the CIA and are running a military campaign against the terrorists. Your most recent undertaking is a mission to find and destroy enemy bunkers in the african jungles. Each bunker you have to attack has a number of weak points, every one of which has to be attacked to destroy that bunker. Your intelligence has a list of the number of weak points per bunker. You fly out with a helicopter to find and destroy these bunkers in the dense jungle. Your only option however to locate these bunkers is to bomb different spots randomly in the jungle until you hit one of the weak points of a bunker. Write a strategic code that can help you minimise the number of bombs you use to destroy these bunkers.

Assumptions

The Jungle can be assumed to be spread out over a 2-d array. Each weak point of a bunker occupies one cell. Weak points can be assumed to be located adjacent to each other in a linear fashion, diagonally or in a square/rectangular fashion in the 2-d array.

Input

The first line would contain an integer n (10 <= n <= 500) denoting dimension of the square grid.
The second line would contain an integer m (m < n*n) denoting the total number of bunkers in the square grid.

Event Rules

The jungle is assumed to be a grid of any size between 10*10 to 500*500.
The weak points of one bunker can be found only in a continuous linear/ diagonal/ square/ rectangular fashion in the rectangular grid. So once you have found one weak point in a bunker, you can rest assured that the next one will be on any of the adjacent/ diagonal cells.
The position of a cell in the grid is defined by a pair of space separated integers “x y” denoting the cell position with row number x and column number y.
Bombs are air-dropped on to a particular cell in the jungle.
Each query is a string consisting of two space-separated integers printed to the STDOUT. The first integer denotes the x-coordinate of the cell position and the second integer the y-coordinate respectively.
The participants code must send queries to determine if the bomb drop was successful.
The simulator counts the number of queries sent.
For each query, the simulator would either return the string “YES” or “NO” answering whether the cell position mentioned in the query contained a weak point to the participant’s STDIN. The participant’s code must read the result of the query as a string from the STDIN.

Scoring

The participant's code with the minimum number of queries wins.
